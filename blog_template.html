The Predatory Nature Of Data Collection
A look at how platforms quietly collect data behind the promises of better user experience.

After learning about digital signatures, certificates, footprints, cookies, and session IDs, I decided to get some hands-on experience. I fired up Burp Suite, searched for my university's website, and received some very interesting results. Here's what happened when I made a simple request to http://ucsc.edu:

Looking at the results, we can see two GET requests and two POST requests. When I visit a webpage, I expect to receive the requested content—HTML, images, and stylesheets. But here, there are hidden requests. First, let's run through a basic overview of how websites function.

How Do Websites Function?

Everything you see on a webpage is just a file. When you visit a website, your browser requests the HTML, CSS, and JavaScript files. These files are then processed locally on your machine to display the webpage.

Dark patterns are UI choices deliberately designed to trick users into actions they wouldn't otherwise take.

What Actually Gets Collected

Beyond the obvious — name, email, location — platforms build detailed behavioral profiles from things you'd never think to protect.

  • Mouse movement and scroll depth
  • Time spent on individual UI elements
  • Device fingerprints across sessions
  • Inferred demographics from browsing patterns

A Practical Example

Here's a simplified version of what a third-party tracker injected into a page might look like. This runs silently on thousands of sites.

javascript
// Third-party tracker fingerprinting example
const fingerprint = {
  screen:    `${screen.width}x${screen.height}`,
  timezone:  Intl.DateTimeFormat().resolvedOptions().timeZone,
  language:  navigator.language,
  platform:  navigator.platform,
  plugins:   [...navigator.plugins].map(p => p.name),
  canvas:    getCanvasHash(), // unique per device
};

// Silently exfiltrated to a remote server
navigator.sendBeacon('https://tracker.example.com/collect',
  JSON.stringify(fingerprint)
);

None of this requires cookies. It runs in the background, requires no permissions, and is almost impossible to detect without inspecting network traffic.

What You Can Do

  1. Use a DNS-level blocker like Pi-hole or NextDNS
  2. Inspect network requests with browser devtools
  3. Prefer services with open privacy policies and audited codebases
  4. Treat "free" services as a data exchange, not a gift
